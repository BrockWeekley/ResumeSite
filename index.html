and these are my principles of development

Design
I focus on simple, reusable, responsive, and accessible design through modern UI libraries and JavaScript frameworks.

In the 21st century, the web has eaten the world. As the greatest source of user interfacing in human history, the internet plays a part in the lives of billions of human beings around the world across billions of devices. When developing a User Interface, the key principles that I focus on are simple, usable, responsive, and accessible. In my extensive experience with both mobile and desktop development, A11Y and RWD principles have been at the forefront of beautiful, business oriented designs with the purpose of adding high value to both internal and external users.

My designing skills include a multitude of JavaScript frameworks such as React, Vue, and Angular. I also have extensive experience with hybrid single codebase tools such as Ionic and React-Native. This, in combination with a knowledge of UI libraries such as Bootstrap and Material, CSS principles such as grid and flexbox, and other more complex styling languages such as SASS, grant me all of the tools and know-how to build an elegant interface for any user.

Data
Modular stateless endpoints are the key to exposing data safely and effectively.

As I focus the aspects of my development on the user, one of the most important features of any application is data integrity, along with data structure. Clean, reliable code is an application feature that, while unseen, is more important than any other. It is a highlight of my development style to create scaleable projects with reusability in mind. This allows me to account for security vulnerabilities and manage data in an industry standard and practical way.

My data design practice often involves Model View Controller and Object Oriented architecture, designing Java apps using Springboot and MySQL. I have heavy experience building REST APIs in both Java and Python. My knack for data structure extends into the front-end with tools such as TypeScript, Wave, Axe, and Lighthouse which allow me to keep code scaleable, manageable, and modular to serve as a portal to data. In my time managing development teams, GitHub has been instrumental in ensuring my teams write the best code they can through pull requests and code review.

Deploy
Continuous Integration, Continuous Deployment is the key to an interactive client and smooth workflow.

In my extensive experience in both internal GitLab pipelines and external GitHub pipelines, I have learned that CICD is integral to the client and the developer experience. Client feedback and satisfaction is greatly increased by seeing regular changes to products, providing the ability for real hands-on interaction. Developer communication is increased when the barrier of "It works on my machine!" is removed. CICD pipelines also further increase code testability and clarity, as linters and unit tests ensure each push contains valid and positive code.

Building custom clouds with Docker containers in kubernetes is one of the ways I have learned to create a scalable and secure environment, as well as using public cloud options for bursting, such as AWS or Google Cloud. This is increasingly important for internal services, as the days of monolithic applications and companies are coming to an end. It is important that software architects like myself build something for the future, rather than something for the day.
