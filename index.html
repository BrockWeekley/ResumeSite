and these are my principles of development

Design
I focus on simple, usable, responsive, and accessible design through UI libraries and JavaScript frameworks.

In the 21st century, the web has eaten the world. As the greatest source of user interfacing in human history, the internet plays a part in the lives of billions of human beings around the world across billions of devices. When developing a User Interface, the key principles that I focus on are simple, usable, responsive, and accessible. In my extensive experience with both mobile and desktop development, A11Y and RWD principles have been at the forefront of beautiful, business oriented designs with the purpose of adding high value to both internal and external users.

My designing skills include a multitude of JavaScript frameworks such as React, Vue, and Angular. I also have extensive experience with hybrid single codebase tools such as Ionic and React-Native. This, in combination with a knowledge of UI libraries such as Bootstrap and Material, CSS principles such as grid and flexbox, and other more complex styling languages such as SASS, grant me all of the tools and know-how to build an elegant interface for any user.

Data
My codebases are clean, scaleable, and feature safe, as well as the codebases of the teams I manage.

As I focus the aspects of my development on the user, one of the most important features of any application is data integrity, along with data structure. Clean, reliable code is an application feature that, while unseen, is more important than any other. It is a highlight of my development style to create scaleable projects with reusability in mind. This allows me to account for security vulnerabilities and manage data in an industry standard and practical way.

My data design practice often involves Model View Controller and Object Oriented architecture, designing Java apps using Springboot and MySQL. I have heavy experience building REST APIs in both Java and Python. My knack for data structure extends likewise into the front-end with tools such as TypeScript, Wave, Axe, and Lighthouse which allow me to keep code scaleable, manageable, and within best practices. In my time managing development teams, GitHub familiarity has been instrumental in ensuring my teams write the best code they can through pull requests and code review.
